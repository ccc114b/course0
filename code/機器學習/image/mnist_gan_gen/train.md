# GAN 訓練演算法（Generative Adversarial Network）

## 原理

生成對抗網路（GAN）的核心是讓兩個神經網路——生成器 G 和判別器 D——進行零和博弈。生成器試圖製造逼真的偽造影像，判別器則試圖區分真實影像與偽造影像。

訓練過程交替更新兩者：先更新判別器（最大化真實與偽造樣本的分類正確率），再更新生成器（最小化判別器對偽造樣本的辨識能力）。

### 演算法步驟

1. 從 MNIST 資料集載入真實影像 x
2. 判別器訓練（固定生成器）：
   a. 計算判別器對真實影像的輸出 D(x)，與全 1 標籤計算 BCE 損失
   b. 從潛在空間取樣 z，生成偽造影像 G(z)
   c. 計算判別器對偽造影像的輸出 D(G(z))，與全 0 標籤計算 BCE 損失
   d. 兩者平均後反向傳播，更新判別器
3. 生成器訓練（固定判別器）：
   a. 重新取樣 z，生成 G(z)
   b. 計算判別器對 G(z) 的輸出，與全 1 標籤計算 BCE 損失（欺騙判別器）
   c. 反向傳播，更新生成器
4. 重複步驟 2-3 直到收斂

### 數學公式

**對抗損失（min-max 遊戲）**:
$$\min_G \max_D V(D, G) = \mathbb{E}_{x \sim p_{\text{data}}}[\log D(x)] + \mathbb{E}_{z \sim p_z}[\log(1 - D(G(z)))]$$

**判別器損失**:
$$\mathcal{L}_D = -\frac{1}{2}\left(\mathbb{E}[\log D(x)] + \mathbb{E}[\log(1 - D(G(z)))]\right)$$

**生成器損失**:
$$\mathcal{L}_G = -\mathbb{E}[\log D(G(z))]$$

## 使用場景

- 生成式建模與影像合成
- 無監督／半監督學習
- 風格轉換、超解析度
- 資料擴增

## 優缺點

| 優點 | 缺點 |
|------|------|
| 生成影像品質高、銳利 | 訓練不穩定，可能不收斂 |
| 生成速度快（一次推論） | 容易模式崩潰（生成樣本單一） |
| 不需要馬可夫鏈式取樣 | 缺乏評估生成品質的客觀指標 |
| 潛在空間有意義的語意結構 | 超參數敏感 |
