# CNN MNIST 手寫數字分類訓練

## 原理

使用簡單的卷積神經網路（CNN）對 MNIST 手寫數字進行分類。採用監督式學習，最小化交叉熵損失來更新網路參數。

為了提高泛化能力，加入了隨機旋轉（±10°）作為資料擴增，以及 Dropout（0.5）作為正則化。

### 網路結構

- Conv1: 1→32 通道, 3×3, ReLU → MaxPool 2×2
- Conv2: 32→64 通道, 3×3, ReLU → MaxPool 2×2
- Flatten: 64×5×5 = 1600
- FC1: 1600 → 128, ReLU, Dropout(0.5)
- FC2: 128 → 10 (類別數)

### 演算法步驟

1. 載入 MNIST 訓練集，套用資料擴增（隨機旋轉）
2. 對每個 batch：
   a. 前向傳播計算輸出 logits
   b. 計算 CrossEntropyLoss（內含 Softmax）
   c. 反向傳播更新權重
   d. 計算訓練準確率
3. 完成 5 個 epoch 後儲存模型權重

### 數學公式

**交叉熵損失**:
$$\mathcal{L} = -\frac{1}{N} \sum_{i=1}^{N} \log \left( \frac{e^{f_{y_i}}}{\sum_{j=1}^{10} e^{f_j}} \right)$$

**卷積層運算**:
$$h^{(l)}_j = \text{ReLU}\left( \sum_{i} h^{(l-1)}_i * W^{(l)}_{ij} + b^{(l)}_j \right)$$

## 使用場景

- 圖片分類任務的基礎範例
- 深度學習入門教學
- 輕量級 OCR 系統的前端

## 優缺點

| 優點 | 缺點 |
|------|------|
| 訓練快速（5 epoch 即可收斂） | 網路結構簡單，無法處理複雜資料 |
| 準確率高（~99% on MNIST） | 對大角度旋轉敏感 |
| 資料擴增提升泛化 | 無批次歸一化，訓練可能不穩定 |
