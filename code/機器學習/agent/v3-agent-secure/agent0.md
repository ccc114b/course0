# Agent with Security — LLM 安全審查 + 目錄存取控制

## 原理

在 v2-agent-xml 的基礎上，增加了**雙層安全防護**：

### 第一層：LLM-based 命令審查（Reviewer）

與 v3-agent-reviewer 相同，使用第二個 LLM 模型審查每個 shell 命令的安全性，基於六條安全原則判斷 SAFE/UNSAFE。

### 第二層：目錄存取控制（Outside Access Check）

新增 `check_outside_access()` 函數，通過正則表達式解析命令中的路徑參數，檢查是否試圖存取當前目錄以外的檔案系統：
- 絕對路徑（如 `/etc/passwd`）
- 相對路徑中的 `..`
- 不屬於 `cwd` 子路徑的絕對路徑

若偵測到外部存取，詢問使用者是否授權，已授權的路徑會加入 `outside_access_granted` 集合避免重複詢問。

### 安全流程

```
命令 → LLM 審查 (SAFE?) → 目錄檢查 (Outside?) → 使用者授權? → 執行
   ↓ UNSAFE                ↓ 安全
  跳過+回報                 跳過+回報 ← 拒絕
```

## 使用場景

- 需防範惡意命令的安全敏感場景
- 多使用者共享的 Agent 服務
- 示範分層安全防護機制

## 優缺點

| 優點 | 缺點 |
|------|------|
| 雙層安全防護 | 審查延遲增加 |
| 目錄存取控制 | 規則式檢測有盲點 |
| 使用者授權機制 | 需使用者互動 |
| 已授權路徑快取 | 無容器隔離 |
