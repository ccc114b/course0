# SAC 推論腳本 (BipedalWalker-v3)

## 原理

本腳本載入訓練好的 SAC (Soft Actor-Critic) 模型檢查點，在 BipedalWalker-v3 環境中進行推論與視覺化展示。SAC 是一種最大熵強化學習演算法，在最大化累積報酬的同時也最大化策略的熵，促使智能體保持探索的多樣性。

推論時可選擇確定性模式（使用平均動作）或隨機模式（從高斯分布抽樣），後者保留了 SAC 對策略隨機性的建模能力。

## 演算法步驟

1. 解析命令列參數（檢查點路徑、回合數、是否錄影等）
2. 載入預訓練的 GaussianActor 網路權重
3. 建立 Gymnasium 環境（可選錄影包裝器）
4. 對每個回合：
   - 重置環境，取得初始觀測
   - 將觀測轉為張量，傳入 Actor 網路取得動作
   - 執行動作，記錄累積報酬，直到回合終止
5. 輸出統計數據（平均、標準差、最小、最大報酬）

## 數學公式

**動作生成 (確定性)**:
$$a = \tanh(\mu) \cdot \text{act\_scale}$$

**動作生成 (隨機)**:
$$a = \tanh(x) \cdot \text{act\_scale}, \quad x \sim \mathcal{N}(\mu, \sigma)$$

其中 $\mu$ 和 $\log\sigma$ 由神經網路輸出，$\sigma$ 被限制在 $[e^{-5}, e^{2}]$ 範圍內。

## 使用場景

- 載入已訓練的 SAC 模型進行視覺化展示
- 錄製智能體行走的影片
- 評估模型在確定性與隨機策略下的效能差異

## 優缺點

| 優點 | 缺點 |
|------|------|
| 支援確定性與隨機兩種推論模式 | 需要事先訓練好的檢查點檔案 |
| 可錄影保存結果 | 僅適用於 SAC 演算法的模型格式 |
| 支援多種裝置 (CUDA/MPS/CPU) | 無即時訓練監控功能 |
