# 閉合形式控制 — CartPole-v1

## 原理

這是一種不使用機器學習的**手動控制策略**，透過觀察杆子的角度與角速度來決定推車的移動方向。其核心邏輯是一組啟發式規則（Heuristic Rules），直接根據狀態變數的符號與閾值來選擇動作，無需任何訓練過程。

CartPole 的狀態包含四個變數：小車位置 $x$、小車速度 $\dot{x}$、杆子角度 $\theta$、杆子角速度 $\dot{\theta}$。此策略只使用後兩者：當杆子向右傾斜且角速度超過閾值時，連續兩次向右推；反之亦然。

## 演算法步驟

1. 初始化環境，取得初始觀測
2. 對每個時間步：
   - 若 $\theta > 0$（杆子向右傾）且 $\dot{\theta} > 0.01$（角速度也向右）→ 向右推兩次
   - 若 $\theta > 0$ 但 $\dot{\theta} \leq 0.01$ → 向左推一次
   - 若 $\theta < 0$（杆子向左傾）且 $\dot{\theta} < -0.01$ → 向左推兩次
   - 若 $\theta < 0$ 但 $\dot{\theta} \geq -0.01$ → 向右推一次
3. 若回合終止則重置環境，記錄步數

## 數學公式

**控制法則**:
$$a_t = \begin{cases}
1 & \text{若 } \theta_t > 0 \land \dot{\theta}_t > 0.01 \\
0 & \text{若 } \theta_t > 0 \land \dot{\theta}_t \leq 0.01 \\
0 & \text{若 } \theta_t < 0 \land \dot{\theta}_t < -0.01 \\
1 & \text{若 } \theta_t < 0 \land \dot{\theta}_t \geq -0.01
\end{cases}$$

其中 $a=0$ 為向左推，$a=1$ 為向右推。

## 使用場景

- 理解 CartPole 問題結構的教學範例
- 作為強化學習演算法的效能基線（Baseline）
- 不需要學習的簡單平衡控制問題

## 優缺點

| 優點 | 缺點 |
|------|------|
| 無需訓練，即寫即用 | 僅適用於簡單環境 |
| 容易理解和實作 | 無法推廣到未見過的狀態 |
| 程式碼極簡 | 沒有學習能力，無法自動優化 |
