# GPT-0 主程式 (main)

## 原理

本檔案是 GPT-0 模型的入口主程式，負責完整的資料載入、預處理、模型訓練與推論流程。它使用命令列引數指定輸入的文本檔案，讀取後建立字符級詞彙表，初始化模型，執行訓練，最後生成新的文本樣本。

### 演算法步驟

1. **載入資料**: 從檔案讀取文本行，支援自動下載範例資料集
2. **建立詞彙表**: 掃描所有文本，收集不重複字符並分配 ID
3. **Tokenize**: 對每個文本進行字符級 tokenize，前後加上 BOS 標記
4. **初始化模型**: 建立所有權重矩陣並隨機初始化
5. **訓練**: 執行指定步數的梯度下降
6. **推論**: 從 BOS 開始逐 token 生成新的文本

### 數學公式

**字符級 Tokenizer**:
$$vocab = \{c_1, c_2, \dots, c_n\},\; BOS = n$$
$$tokenize(text) = [BOS, id(c_1), id(c_2), \dots, BOS]$$

**UTF-8 編碼長度判斷**:
- 0xxxxxxx: 1 byte
- 110xxxxx: 2 bytes
- 1110xxxx: 3 bytes
- 11110xxx: 4 bytes

### 使用場景

- 文本資料的訓練與生成應用的入口點
- 示範完整的機器學習 pipeline（資料 → 訓練 → 推論）
- 可搭配不同語言的文本資料進行實驗

### 優缺點

| 優點 | 缺點 |
|------|------|
| 完整的 pipeline 實作，一鍵訓練與推論 | UTF-8 處理邏輯較複雜 |
| 支援 UTF-8 多字元編碼 | 無錯誤處理與中斷恢復機制 |
| 資料集自動下載，開箱即用 | 訓練參數寫死，需修改原始碼才能調整 |
