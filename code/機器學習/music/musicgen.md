# 馬可夫鏈旋律生成器

## 原理

本程式使用**二階馬可夫鏈（Second-Order Markov Chain）** 對旋律進行建模與生成。馬可夫鏈是一種隨機過程，其中未來狀態的轉移機率僅依賴於當前狀態（馬可夫性質）。二階模型則擴展為依賴前兩個音符來預測下一個音符，因此具有更強的旋律連貫性。

轉移機率從 `markov.txt` 中讀取，每條記錄格式為 `前兩個音符 下一個音符 權重`，權重越高表示該轉移越可能被選中。

### 演算法步驟

1. **載入轉移資料**：讀取 `markov.txt`，以 `(n1, n2)` 為鍵（狀態），`n3` 為候選音符，權重作為抽樣依據。
2. **隨機初始化**：從所有狀態中隨機選取一個起始狀態 `(n1, n2)` 作為旋律的前兩個音。
3. **反覆生成**：根據當前狀態的轉移機率抽樣下一個音符，並滑動視窗更新狀態。
4. **長度控制**：若抽到 `END` 記號且旋律已達最低長度門檻則停止；否則跳過 `END` 或隨機跳轉。
5. **死胡同處理**：若當前狀態無對應轉移，則隨機跳轉到新狀態。

### 數學公式

**二階馬可夫鏈的轉移機率**:
$$P(X_t = x_t \mid X_{t-1}=x_{t-1}, X_{t-2}=x_{t-2}) = \frac{w(x_{t-2}, x_{t-1}, x_t)}{\sum_{x'} w(x_{t-2}, x_{t-1}, x')}$$

其中 $w$ 為該轉移的權重。

**權重抽樣（Weighted Random Sampling）**:
$$P(\text{選擇第 } i \text{ 個候選}) = \frac{w_i}{\sum_{j} w_j}$$

## 使用場景

- 自動音樂作曲原型
- 遊戲中的動態背景音樂生成
- 教學用途：展示馬可夫鏈在序列生成中的應用

## 優缺點

| 優點 | 缺點 |
|------|------|
| 實作簡單，直觀易懂 | 二階模型捕捉長期結構能力有限 |
| 權重系統可靈活控制風格 | 依賴人工編寫的轉移規則 |
| 每次生成結果不同，具多樣性 | 無全局結構規劃（無 ABA 等曲式） |
| 長度可控制 | 無法處理未見過的狀態組合 |
