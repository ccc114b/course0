# AlphaZero 五子棋人機對戰

## 原理

本程式載入以 **AlphaZero** 類架構訓練好的五子棋模型，提供人類 vs 電腦或電腦 vs 電腦的對戰介面。AlphaZero (Silver et al., 2017) 的核心是結合**深度神經網路**與**蒙地卡羅樹搜尋（MCTS）**，神經網路輸出「局面價值」與「候選動作機率」，引導 MCTS 的搜尋方向。

對戰時，AI 使用已訓練好的 `AlphaZeroNet` 模型搭配 MCTS（400 次模擬），不作隨機探索，直接選取 `π` 機率最高的動作。

### 演算法步驟

1. **解析參數**：接收兩個命令列參數，指定先手與後手為人類 (`P`) 或電腦 (`C`)。
2. **載入模型**：讀取 `alphazero_gomoku.pth`（若不存在則使用未訓練的隨機模型）。
3. **建立 MCTS**：以 `num_simulations=400` 進行搜尋，`c_puct=1.0` 控制探索。
4. **遊戲迴圈**：
   - 人類回合：由命令列輸入行列座標。
   - AI 回合：將當前棋盤以「當前玩家視角」轉換後送入 MCTS，得到動作機率 `π`，取最大值對應動作。
5. **視角轉換**：MCTS 搜尋時固定以「先手 (1)」視角進行，因此需將棋盤乘以 `player_turn`（1 或 -1）來轉換。
6. **勝負判定**：每步落子後檢查是否連線或棋盤已滿。

### 數學公式

**MCTS 動作選擇（PUCT 演算法）**:
$$a_t = \arg\max_a \left( Q(s, a) + c_{\text{puct}} \cdot P(s, a) \cdot \frac{\sqrt{\sum_b N(s, b)}}{1 + N(s, a)} \right)$$

其中 $Q(s,a)$ 為平均回訪價值，$P(s,a)$ 為神經網路輸出的先驗機率，$N(s,a)$ 為訪問次數。

**神經網路輸出**:
$$(p, v) = f_\theta(s)$$

$p$ 為各動作的先驗機率分佈，$v$ 為當前局面預期勝率（從當前玩家視角，範圍 [-1, 1]）。

## 使用場景

- 展示 AlphaZero 訓練成果的對戰介面
- 人機對弈學習與娛樂
- AI vs AI 測試不同 checkpoint 的強度

## 優缺點

| 優點 | 缺點 |
|------|------|
| 支援彈性的人機 / 機機對戰 | 無圖形化介面（純文字） |
| 視角轉換設計簡潔 | 依賴 train.py 中的架構定義 |
| 可使用不同模擬次數調整強度 | 無時間控制機制 |
