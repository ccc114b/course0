# 有限狀態機 (Finite State Machine, FSM)

**標籤**: #有限狀態機 #FSM #DFA #NFA #正則語言 #自動機理論 #計算理論
**日期**: 2024-01-15
**更新**: 2026-04-27

有限狀態機（Finite State Machine，簡稱 FSM），又稱有限狀態自動機（Finite State Automaton，FSA），是計算理論中最基本也是最重要的計算模型之一。它用於描述一個系統在有限數量的狀態之間進行轉換的行為，根據輸入字元從一個狀態轉移到另一個狀態。有限狀態機在電腦科學的許多領域都有廣泛應用，包括編譯器的詞彙分析、協定設計、數位電路設計、遊戲 AI 等。

## 發展歷程

### 理論起源 (1950年代)

有限狀態機的概念起源於 1950 年代，當時主要應用於類神經網路和開關電路的研究。Mealy 和 Moore 分別在 1955 年和 1956 年提出了兩種著名的有限狀態機模型，至今仍是數位電路設計的基礎。

Mealy 機器的輸出與當前狀態和輸入都有關，而 Moore 機器的輸出只與當前狀態有關。這兩種模型各有優點，在不同應用場景中發揮作用。

### 在編譯器中的應用 (1960年代-1970年代)

肯·湯普森（Ken Thompson）在 1968 年開發的 Unix 編輯器 ed 和隨後的 grep 工具，首次將有限狀態機與正則表達式結合，用於模式匹配。這一創新後來被應用於 lex 等詞彙分析器生成工具，奠定了編譯器詞法分析的理論基礎。

### 現代應用 (1980年代至今)

有限狀態機在軟體工程中持續發揮重要作用，特別是在協定實現、遊戲開發、工作流程引擎、數位電路設計等領域。

## 形式定義

有限狀態機是一個五元組 M = (Q, Σ, δ, q₀, F)，各元件的意義如下：

```
Q  - 有限的非空狀態集合 (States)
Σ  - 有限的輸入字母表 (Alphabet)
δ  - 轉移函數 (Transition Function): Q × Σ → Q
q₀ - 起始狀態 (Initial State), q₀ ∈ Q
F  - 接受狀態集合 (Final/Accepting States), F ⊆ Q
```

根據轉移函數的不同特性，有限狀態機可分為兩種主要類型：

1. **確定型有限狀態機 (DFA)**：轉移函數 δ(q, a) 至多返回一個狀態
2. **非確定型有限狀態機 (NFA)**：轉移函數 δ(q, a) 可以返回多個狀態，甚至允許空轉移 (ε-轉移)

## 確定型有限狀態機 (DFA)

在 DFA 中，每個狀態對每個輸入符號只有唯一的下一個狀態。這意味著對於相同的輸入，總是產生相同的轉移路徑。

**DFA 的特點**：
- 每個狀態對每個輸入符號只有一個轉移
- 沒有 ε-轉移
- 任意時刻只有一個活躍狀態
- 空間需求較小，執行行為確定

**DFA 形式化定義**：

```
DFA = (Q, Σ, δ, q₀, F)

其中：
- Q = {q0, q1, q2, ...}  # 狀態集合
- Σ = {a, b, c, ...}      # 輸入字母表
- δ : Q × Σ → Q           # 轉移函數
- q0 ∈ Q                  # 起始狀態
- F ⊆ Q                  # 接受狀態集合
```

### 範例：識別以 01 結尾的二進位字串

讓我們設計一個 DFA 來識別所有以 "01" 結尾的二進位字串：

- 狀態 q0：起始狀態，還沒看到有效的結尾
- 狀態 q1：剛讀到 '0'
- 狀態 q2：已讀到 "01"（接受狀態）

轉移函數：
- δ(q0, 0) = q1, δ(q0, 1) = q0
- δ(q1, 0) = q1, δ(q1, 1) = q2
- δ(q2, 0) = q1, δ(q2, 1) = q2

接受狀態：F = {q2}

這個 DFA 會接受 "01", "001", "101", "0101" 等，拒絕 "0", "1", "10", "100" 等。

## 非確定型有限狀態機 (NFA)

在 NFA 中，一個狀態對同一個輸入符號可能有多個可能的轉移，也可能沒有轉移。此外，NFA 允許 ε-轉移，即不需要消耗輸入字元就可以進行的轉移。

**NFA 的特點**：
- 一個狀態對同一個輸入可能有多個下一狀態
- 允許 ε-轉移（空字元轉移）
- 可能需要指數級的空間來模擬
- 設計相對直觀，描述能力與 DFA 等價

**NFA 形式化定義**：

```
NFA = (Q, Σ, δ, q₀, F)

其中：
- Q = {q0, q1, q2, ...}         # 狀態集合
- Σ = {a, b, c, ...}             # 輸入字母表
- δ : Q × Σ ∪ {ε} → P(Q)         # 轉移函數（返回狀態的集合）
- q0 ∈ Q                         # 起始狀態
- F ⊆ Q                          # 接受狀態集合
```

## DFA 與 NFA 的比較

| 特性 | DFA | NFA |
|------|-----|-----|
| 轉移確定性 | 每個狀態每個輸入只有一個轉移 | 可能有多個轉移 |
| ε-轉移 | 無 | 有 |
| 記憶體需求 | O(1) | 可能需指數空間 |
| 設計難度 | 較高 | 較低 |
| 等價性 | 與 NFA 等價 | 與 DFA 等價 |
| 接受語言 | 正則語言 | 正則語言 |

## 等價性證明

DFA 和 NFA 接受的是同一類語言——正則語言。這個結論基於以下定理：

**定理**：對於任意 NFA，存在一個等價的 DFA 接受相同的語言。

**證明思路（子集構造法）**：
1. DFA 的每個狀態對應 NFA 狀態集合的一個子集
2. DFA 的起始狀態是包含 NFA 起始狀態的集合的 ε-閉包
3. 對於狀態集合 S 和輸入 a，DFA 轉移到所有從 S 中的狀態經過 a 可達的狀態的 ε-閉包
4. 如果狀態集合包含任何 NFA 接受狀態，則 DFA 的對應狀態為接受狀態

這個構造可能產生指數數量的 DFA 狀態，但產生的 DFA 總是正確的。

## 有限狀態機的極小化

### 等價狀態

兩個狀態稱為等價，如果：
1. 從其中一個狀態出發，讀取任何字串到達的都是接受/拒絕狀態
2. 從另一個狀態出發也一樣

### Myhill-Nerode 定理

Myhill-Nerode 定理提供了判斷兩個狀態是否等價的框架，也給出了正則語言的特徵：

**定義**：對於語言 L，狀態 p 和 q 是不可區分的，如果對於所有字串 w，pw ∈ L ⇔ qw ∈ L。

**定理**：語言 L 是正則的當且僅當 L 的 Myhill-Nerode 等價類的數量是有限的。

這意味著：語言是正則的 iff 我們可以用有限狀態機來識別它。

## 有限狀態機的限制

有限狀態機雖然強大，但有其無法解決的問題。它只能識別正則語言，無法識別需要「記憶」或「計數」的語言。

### FSM 無法識別的語言

1. **aⁿbⁿ** (n ≥ 1)：需要記憶讀取了多少個 'a'
2. **匹配的括號**：需要棧來追蹤括號深度
3. **質數個 'a'**：需要無限記憶體

這些語言需要圖靈機或更強大的計算模型才能識別。

## 應用領域

1. **詞彙分析器 (Lexer)**：編譯器的第一階段，使用正則表達式 → NFA → DFA 的流程來識別 tokens
2. **協定狀態機**：TCP 協定使用複雜的有限狀態機來管理連線狀態
3. **數位電路設計**：時序邏輯電路的基礎模型
4. **遊戲 AI**：簡單的敵人行為狀態機
5. **工作流程引擎**：業務流程的狀態建模

## 舉例說明

### 舉例：設計 DFA 識別以 "01" 結尾的二進位字串

狀態定義：
- q₀：起始狀態，還沒看到有效結尾
- q₁：剛讀到 '0'
- q₂：已讀到 "01"（接受狀態）

轉移函數：
- δ(q₀, '0') = q₁，δ(q₀, '1') = q₀
- δ(q₁, '0') = q₁，δ(q₁, '1') = q₂
- δ(q₂, '0') = q₁，δ(q₂, '1') = q₂

測試：
- "01" → q₀ --0→ q₁ --1→ q₂（接受）✓
- "1101" → q₀--1→ q₀--1→ q₀--0→ q₁--1→ q₂（接受）✓
- "10" → q₀--1→ q₀--0→ q₁（拒絕）✗

### 舉例：NFA 的設計

設計 NFA 識別包含子字串 "ab" 或 "ba" 的字串。

優點：NFA 可以並行探索兩條路徑。

```
       a              b
q₀ ──────────▶ q₁    q₀ ──────────▶ q₂
     ε                     ε
      \                   /
       ◀─────────────────◀
              ε
```

只要有一條路徑到達接受狀態，即接受輸入。

### 舉例：DFA 與 NFA 的等價性

NFA：
- 狀態：{q₀, q₁, q₂}
- 字母表：{0, 1}
- 起始狀態：q₀
- 接受狀態：{q₂}
- 轉移：
  - δ(q₀, '0') = {q₀, q₁}
  - δ(q₀, '1') = {q₀}
  - δ(q₁, '1') = {q₂}

轉換為 DFA（子集構造）：
- 起始：ε-閉包({q₀}) = {q₀}
- 讀 '0'：{q₀, q₁}
- 讀 '1'：{q₀}

DFA 狀態對應 NFA 狀態集合，這展示了兩者的等價性。

### 舉例：FSM 無法識別的語言

語言 L = {0ⁿ1ⁿ | n ≥ 1}

為什麼 FSM 無法識別？
- FSM 只有有限個狀態
- 無法「記住」讀到了多少個 0
- 當讀到 1 時，無法判斷數量是否匹配

例如：DFA 讀 "000111" 和 "0000111" 都需要進入相同狀態，但兩者一個接受、一個拒絕。

這需要下推自動機（使用堆疊）才能識別。
