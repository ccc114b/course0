# 計算理論

計算理論是電腦科學的理論基礎，探討計算的本質、能力與極限。這個領域回答了「什麼可以被計算？」、「計算需要多少資源？」以及「哪些問題是無法解決的？」等基本問題。計算理論的發展始於 1930 年代，當時數學家們試圖理解希爾伯特提出的判定問題，最終形成了圖靈機、λ演算等計算模型，為現代電腦科學奠定了堅實的理論基礎。

## 計算模型層次

計算理論建立了不同層次的計算模型，每個模型都有其獨特的能力和限制。從最強大的圖靈機到最簡單的正則語言，這些模型構成了一個清晰的階層結構，反映了計算能力的漸進增強。

```
┌─────────────────────────────────────────────────────────────┐
│                      計算模型層次                             │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  圖靈機 (Turing Machine)                                    │
│  • 最強大的計算模型                                         │
│  • 可模擬任何其他合理計算模型                               │
│  • Church-Turing 論點的基礎                                 │
│  ↓                                                           │
│  λ演算 (Lambda Calculus)                                    │
│  • 函數式計算的基礎                                         │
│  • 閉包、高階函數                                           │
│  ↓                                                           │
│  上下文無關文法 (Context-Free Grammar)                      │
│  • 程式語言語法理論                                         │
│  • 自動機理論                                               │
│  ↓                                                           │
│  正規語言 (Regular Language)                                │
│  • 有限狀態機                                               │
│  • 詞法分析基礎                                             │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## 圖靈機

圖靈機由艾倫·圖靈於 1936 年提出，是計算理論的核心概念。它是一個理論模型，由無限期性的磁帶、讀寫頭和有限狀態控制器組成。圖靈機雖然是一個抽象模型，但它精確地定義了什麼是可計算的，為後來所有電腦的設計提供了理論依據。

### 形式定義

```
圖靈機 M = (Q, Σ, Γ, δ, q₀, q_accept, q_reject)

其中：
• Q: 有限狀態集合
• Σ: 輸入字母表（不含空白）
• Γ: 磁帶字母表（包含空白 □）
• δ: 轉移函數 Q × Γ → Q × Γ × {L, R}
• q₀: 起始狀態
• q_accept: 接受狀態
• q_reject: 拒絕狀態
```

### 運作方式

圖靈機的運作方式可以描述如下：首先，讀寫頭讀取目前磁帶格子上的符號；然後，根據目前狀態和讀到的符號，轉移函數決定下一個狀態、寫入的新符號，以及讀寫頭移動方向；最後，重複直到進入接受或拒絕狀態。

例如，識別語言 {0ⁿ1ⁿ | n ≥ 1} 的步驟如下：從最左邊開始，將 0 改為 X 並向右移動；尋找對應的 1 並改為 Y，然後向左移動；重複直到處理完所有 0；若成功配對則接受輸入。

### Church-Turing 論點

Church-Turing 論點是計算理論中最基本的假設之一。這個論點聲稱：任何可有效計算的函數都可以由圖靈機計算。這不是一個定理，而是一個猜想，它基於丘奇從 λ演算角度和圖靈從圖靈機角度的獨立研究，兩者被證明是等價的。

這個論點的意義深遠：圖靈機定義了「可計算」的數學意義；它意味著之後沒有任何計算模型能超越圖靈機的計算能力；這為整個電腦科學提供了理論基石。

## λ演算

λ演算由阿隆佐·丘奇在 1936 年提出，是函數式程式語言的理論基礎。與圖靈機不同，λ演算從函數的角度描述計算，強調函數的應用和代換。

### 語法

λ演算的語法非常簡潔，只包含三種表達式：

```
λ calculus 語法：
<表達式> ::= <變數>
           | (λ<變數>.<表達式>)
           | (<表達式><表達式>)

範例：
• x                - 變數
• λx.M             - 函數（x 為參數，M 為函式體）
• (λx.M N)         - 函數應用（將 N 應用於 λx.M）
```

### 核心概念

λ演算中有三個重要的轉換規則：

α轉換（Alpha Conversion）允許更改函數的參數名稱而不改變其語義，例如 λx.M 等價於 λy.M[x/y]。

β歸約（Beta Reduction）是函數應用的核心規則，(λx.M) N 表示將 N 替換 M 中的 x 得到 M[x/N]。

η轉換（Eta Conversion）反映了函數的 extensionality，λx.(M x) 等價於 M（前提是 x 不在 M 中自由出現）。

### Y Combinator

Y Combinator 是實現遞迴的關鍵，它允許在不具名遞迴的情況下定義遞迴函數。Y Combinator 的定義是：

```
Y = λf.(λx.f (x x)) (λx.f (x x))
```

這使得我們可以定義諸如階層函數這樣的遞迴函數：fact = Y (λf n. if n==0 then 1 else n * f(n-1))

## 形式文法

形式文法描述語言的語法結構，是編譯器理論的基礎。形式文法由四個元素組成：非終結符集合、終結符集合、產生式規則集合，以及起始符號。

### Chomsky 階層

Chomsky 階層是形式文法最重要的分類系統，將文法分為四個層次：

```
┌─────────────────────────────────────────────────────────────┐
│                      Chomsky 階層                           │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  Type 0: 無限制文法 (Phrase Structure)                     │
│  • 產生式：α → β，α, β 為任意字串                          │
│  • 對應：圖靈機                                             │
│  • 語言：遞迴可枚舉語言                                     │
│                                                              │
│  Type 1: 上下文敏感文法 (Context-Sensitive)                │
│  • 產生式：αAβ → αγβ                                       │
│  • 對應：線性有界自動機                                     │
│  • 語言：上下文敏感語言                                     │
│                                                              │
│  Type 2: 上下文無關文法 (Context-Free)                     │
│  • 產生式：A → γ                                            │
│  • 對應：下推自動機                                         │
│  • 語言：大多數程式語言語法                                 │
│                                                              │
│  Type 3: 正規文法 (Regular)                                 │
│  • 產生式：A → aB 或 A → a                                 │
│  • 對應：有限狀態機                                         │
│  • 語言：正則表達式描述的語言                               │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 語法分析

上下文無關文法可以用於描述程式語言語法。例如，算術表達式的文法可以定義為：

```
E → T | E + T
T → F | T * F
F → ( E ) | id
```

導出範例：E → T → F * F → id * F → id * id

這個導出過程對應的分析樹展示了如何將一個句子分解為其語法組成部分。

## 舉例說明

### 舉例：DFA 識別二進位偶數

設計一個 DFA 識別所有以偶數個 1 結尾的二進位字串。

- 狀態 q_even：目前為止讀到偶數個 1
- 狀態 q_odd：目前為止讀到奇數個 1
- 起始狀態：q_even（空字串視為 0 個 1，偶數）
- 接受狀態：q_even

轉移函數：
- 讀到 0：狀態不變
- 讀到 1：奇偶翻轉

因此，輸入 "1101"（結尾是 1，奇數個 1）會被拒絕；"11010"（結尾是 0，偶數個 1）會被接受。

### 舉例： Chomsky 文法產出字串

使用正規文法 S → aS | bS | ε 產出 "aba"：
S → aS → abS → abaS → aba

使用上下文無關文法 S → aSb | ε 產出 "aabb"：
S → aSb → aaSbb → aabb

### 舉例：多項式時間 vs 指數時間

若 n = 10：
- O(n) = 10 步
- O(n²) = 100 步
- O(2ⁿ) = 1024 步

若 n = 30：
- O(n) = 30 步
- O(n²) = 900 步
- O(2³⁰) ≈ 10 億步

這就是為什麼 NP-Complete 問題被認為「困難」——隨著輸入增大，所需時間爆炸性增長。

### 舉例：停機問題的反證證明

假設存在函式 Halt(P, I) 能判斷任意程式 P 對輸入 I 是否會停止。構造：

```
Paradox(P):
    if Halt(P, P) then
        無限迴圈
    else
        停止
```

現在問：Paradox(Paradox) 會停機嗎？
- 若 Halt 說會停機 → Paradox 會進入無限迴圈（不停機）
- 若 Halt 說不會停機 → Paradox 會停止（停機）

矛盾！因此 Halt 不可能存在。

## 有限狀態機

有限狀態機是最簡單的計算模型，用於識別正則語言。它在編譯器的詞法分析、協定設計、數位電路設計等領域有廣泛應用。

### 確定性 vs 非確定性

確定型有限狀態機（DFA）的每個狀態對每個輸入只有一個轉移，而非確定型有限狀態機（NFA）的一個狀態對同一輸入可以有多個轉移，並允許空轉移（ε）。

DFA 的形式化定義為 M = (Q, Σ, δ, q₀, F)，其中 Q 是狀態集合，Σ 是輸入字母表，δ 是轉移函數 Q × Σ → Q，q₀ 是起始狀態，F 是接受狀態集合。

NFA 的形式化定義類似，但轉移函數為 δ : Q × Σ ∪ {ε} → P(Q)，返回狀態的集合。

值得注意的是，DFA 和 NFA 是等價的——對於任何 NFA，都存在一個等價的 DFA 接受相同的語言。這個結論通過子集構造法可以證明。

## 可計算性理論

可計算性理論探討哪些問題可以被解決，哪些問題是無法解決的。

### 停機問題

停機問題是計算理論中最著名的不可判定問題。它詢問：是否存在一個程式能判斷任意程式是否會停止？

證明使用反證法：假設存在函式 Halt(P, I) 判斷程式 P 對輸入 I 是否會停止。我們構造函式 Paradox(P)，當 Halt(P, P) 為真時進入無限迴圈，否則停止。呼叫 Paradox(Paradox) 會導致矛盾——如果 Halt 說它會停止，它實際上會進入無限迴圈；如果 Halt 說它會停止，它實際上會停止。因此這樣的 Halt 函式不可能存在。

停機問題的意義在於：存在無法用演算法解決的問題；電腦有本質上的限制；數學真理不等於可計算真理。

### 不可判定問題

除了停機問題，還有許多經典的不可判定問題，包括：Post 對應問題、一階邏輯有效性、上下文無關文法交集、DFA 語言等價性，以及萬有歸納。

Rice 定理更進一步指出：任何非平凡的語言性質（即非語言本身，亦非所有語言）都是不可判定的。

## 複雜度理論

複雜度理論將問題按照求解所需的資源（如時間、空間）進行分類。

### 計算複雜度

時間複雜度類別按照增長速率分為多個層次：

```
┌─────────────────────────────────────────────────────────────┐
│                    複雜度層次                                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  O(1)       常數時間     陣列存取                          │
│      ↓                                                   │
│  O(log n)   對數時間     二分搜尋                          │
│      ↓                                                   │
│  O(n)       線性時間     線性搜尋                          │
│      ↓                                                   │
│  O(n log n) 線性對數     合併排序                         │
│      ↓                                                   │
│  O(nᵏ)      多項式時間   P 類                             │
│      ↓                                                   │
│  O(kⁿ)      指數時間     組合爆炸                        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

空間複雜度類別包括：O(1) 常數空間、O(n) 線性空間、PSPACE 多項式空間，以及 EXPTIME 指數時間。

### P vs NP

P 類問題包含所有可以在多項式時間內由確定性圖靈機解決的問題，例如排序、搜尋、最短路徑等。

NP 類問題包含所有可以在多項式時間內驗證解答的問題。例如，給定一個布爾公式和一組變數指派，我們可以很快驗證這個指派是否滿足公式；給定一條路徑，我們可以很快驗證它是否為哈密頓回路。

NP-Complete 是 NP 類中最難的問題，任何 NP 問題都可以在多項式時間內歸約到 NP-Complete 問題。著名的 NP-Complete 問題包括：SAT、3-SAT、頂點覆蓋、哈密頓回路、旅遊推銷員問題、子集和、團隊分割、圖著色、背包問題等。

P vs NP 問題是計算機科學中最大的未解決問題，詢問是否所有可以在多項式時間內驗證的問題都可以在多項式時間內解決。目前大多數人相信 P ≠ NP，但這尚未被證明。
