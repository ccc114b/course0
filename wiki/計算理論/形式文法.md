# 形式文法 (Formal Grammar)

**標籤**: #形式文法 #FormalGrammar #CFG #Chomsky #文法階層 #編譯器
**日期**: 2024-01-15
**更新**: 2026-04-27

形式文法是描述語言語法的數學工具，定義了如何產生合法的字串。形式文法是編譯器理論的基礎，用於精確地描述程式語言的語法結構，是語法分析的理論依據。

## 歷史背景

形式文法的研究始於 1950 年代，諾姆·喬姆斯基（Noam Chomsky）在語言學研究中首次提出了形式文法的概念。喬姆斯基試圖用數學方法描述自然語言的語法結構，這一努力最終產生了著名的 Chomsky 階層，將形式文法分為四個層次，每個層次對應不同的計算能力。

形式文法與自動機理論密切相關。喬姆斯基發現，形式文法的類型與識別語言的自動機類型之間存在一一對應關係，這為編譯器的自動生成提供了理論基礎。

## 形式定義

形式文法是一個四元組 G = (N, Σ, P, S)，各元件的意義如下：

```
G = (N, Σ, P, S)

N  - 非終結符集合 (Non-terminals)
     表示語法類別，如「句子」、「名詞片語」等
     
Σ  - 終結符集合 (Terminals)
     表示語言中的基本符號，如關鍵字、運算子等
     
P  - 產生式規則集合 (Productions)
     定義如何從非終結符產生字串的規則
     
S  - 起始符號 (Start Symbol)
     開始推導的頂層非終結符，S ∈ N
```

### 產生式規則

產生式規則的形式為 α → β，其中 α 和 β 是由終結符和非終結符組成的字串。產生式表示「左邊可以推導出右邊」。

範例：
```
S → aSb        # S 可以推導出 aSb
S → ε          # S 可以推導出空字串
```

## Chomsky 語言階層

Chomsky 階層是形式文法最重要的分類系統，將文法分為四個層次，每個層次具有不同的表達能力和計算複雜度：

```
┌─────────────────────────────────────────────────────────────┐
│                      Chomsky 階層                           │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  Type 0: 無限制文法 (Phrase Structure Grammar)            │
│  • 產生式：α → β，α, β 為任意字串（α ≠ ε）              │
│  • 對應：圖靈機                                            │
│  • 語言：遞迴可枚舉語言                                    │
│  • 能力：最強，可表達任何可計算的語言                       │
│                                                              │
│  Type 1: 上下文敏感文法 (Context-Sensitive Grammar)       │
│  • 產生式：αAβ → αγβ                                      │
│    （非終結符 A 在上下文 α...β 中可被替換為 γ）           │
│  • 對應：線性有界自動機                                    │
│  • 語言：上下文敏感語言                                    │
│  • 特性：推導過程中字串長度不減少                          │
│                                                              │
│  Type 2: 上下文無關文法 (Context-Free Grammar, CFG)       │
│  • 產生式：A → γ                                           │
│    （非終結符 A 可被替換為任意字串 γ）                    │
│  • 對應：下推自動機 (Pushdown Automaton)                  │
│  • 語言：上下文無關語言                                    │
│  • 應用：大，大多數程式語言語法                            │
│  • 特性：可以用 BNF 表示法描述                            │
│                                                              │
│  Type 3: 正規文法 (Regular Grammar)                        │
│  • 產生式：A → aB 或 A → a                                 │
│    （右線性：非終結符只在右側出現）                        │
│    或 A → Ba 和 A → a                                      │
│    （左線性：非終結符只在左側出現）                        │
│  • 對應：有限狀態機                                        │
│  • 語言：正則語言                                          │
│  • 應用：正則表達式、詞法分析                              │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## 上下文無關文法 (CFG)

上下文無關文法是程式語言語法定義的基礎。CFG 的產生式左邊只有單個非終結符，這意味著非終結符的替換不依賴於其上下文。

### 範例：算術表達式

算術表達式可以用以下 CFG 描述：

```
E → T               # 表達式 → 項
E → E + T           # 表達式 → 表達式 + 項
T → F               # 項 → 因子
T → T * F           # 項 → 項 * 因子
F → ( E )           # 因子 → ( 表達式 )
F → id              # 因子 → 識別符
```

這個文法定義了算術表達式的優先級和結合性。

### 導出過程

使用上述文法導出表達式 "id + id * id" 的過程：

```
E  → E + T
   → T + T
   → F + T
   → id + T
   → id + T * F
   → id + F * F
   → id + id * F
   → id + id * id
```

### 分析樹

導出過程可以表示為一棵分析樹（Parse Tree）：

```
        E
       /|\
      / | \
     E  +  T
     |     /|\
     T    / | \
     |   T  *  F
     F   |     |
     |   F     id
    id  id
```

分析樹直觀地展示了語法結構，是語法分析的基礎。

## 正規文法

正規文法是 最簡單的文法類型，其產生式規則非常受限，只能產生正則語言。正規文法與有限狀態機、正則表達式等價。

### 範例：識別以 01 結尾的二進位字串

```
S → 0A
S → 1S
A → 1B
B → 0B
B → 1S
B → ε
```

這個文法產生所有以 "01" 結尾的二進位字串。

### 與正則表達式的對應

正規文法產生的語言可以用正則表達式描述，反之亦然。例如：
- 文法規則 A → aB, B → b 對應正則表達式 ab
- 文法規則 A → aA | ε 對應正則表達式 a*

## 語法分析

語法分析是編譯器的核心階段，負責根據文法規則分析輸入的 token 序列是否合法，並構建分析樹。

### 自上而下分析

從起始符號開始，反覆應用產生式，直到產生輸入字串。常見方法包括：
- 遞歸下降分析
- LL(1) 分析

### 自下而上分析

從輸入字串開始，反覆進行「歸約」直到只剩起始符號。常見方法包括：
- LR(1) 分析
- SLR, LALR 分析

### LL vs LR

| 方法 | 策略 | 優點 | 缺點 |
|------|------|------|------|
| LL(1) | 自上而下 | 簡單、直觀 | 不能處理左遞迴 |
| LR(1) | 自下而上 | 能力強大 | 建表複雜 |

## 與自動機的對應

形式文法與自動機之間存在深刻的對應關係：

```
┌─────────────────────────────────────────────────────────────┐
│  文法與自動機的對應                                         │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  正規文法 ←→ 有限狀態機 ←→ 正則表達式                      │
│      ↓                                                       │
│  上下文無關文法 ←→ 下推自動機                              │
│      ↓                                                       │
│  上下文敏感文法 ←→ 線性有界自動機                          │
│      ↓                                                       │
│  無限制文法 ←→ 圖靈機                                      │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

這種對應關係意味著：我們可以用文法來描述語言，用自動機來識別語言，兩者是等價的表達方式。

## 實際應用

形式文法在電腦科學的許多領域有重要應用：

1. **程式語言語法**：C、Java、Python 等語言的語法都是用 CFG 定義的
2. **標記語言**：JSON、XML 的結構可以用文法描述
3. **資料庫查詢**：SQL 語法可以用文法描述
4. **網路協定**：HTTP、TCP 等協定的狀態可以用文法建模
5. **自然語言處理**：雖然自然語言不完全是上下文無關的，但文法仍是重要工具

## 舉例說明

### 舉例：正規文法產生語言

文法 G = (N, Σ, P, S)，N = {S}，Σ = {a, b}，P = {S → aS, S → bS, S → ε}

產生過程：
```
S → aS
  → abS
  → abaS
  → ababS
  → abab
```

這個文法產生所有由 a 和 b 組成的字串（a*b*）。

對應的正則表達式：a*b*

### 舉例：上下文無關文法產生算術表達式

文法：
- E → E + T | T
- T → T * F | F
- F → ( E ) | id

導出 "id + id * id"：

```
E → E + T
  → T + T
  → F + T
  → id + T
  → id + T * F
  → id + F * F
  → id + id * F
  → id + id * id
```

分析樹：
```
      E
    / | \
   E  +  T
   |    /|\
   T   T * F
   |   |   |
   F   F   id
   |
  id
```

### 舉例：上下文敏感文法產生 aⁿbⁿcⁿ

文法 G：
- S → aBC | aSBC
- CB → CZ
- CZ → WZ
- WZ → WC
- WC → BC
- aB → ab
- bB → bb
- bC → bc
- cC → cc

產生 "aabbcc"：
```
S → aSBC → aaSBCBC → aaaBCBCBC → aaaBCBCBC（無法繼續...）
```

這是一個上下文相關文法，因為產生式需要考慮上下文。

### 舉例：LL(1) 分析法

文法：
- E → T E'
- E' → + T E' | ε
- T → F T'
- T' → * F T' | ε
- F → ( E ) | id

分析輸入 "id + id * id"：

| 棧 | 輸入 | 動作 |
|---|------|------|
| E $ | id + id * id $ | E → T E' |
| T E' $ | id + id * id $ | T → F T' |
| F T' E' $ | id + id * id $ | F → id |
| id T' E' $ | id + id * id $ | 匹配 id |
| T' E' $ | + id * id $ | T' → ε |
| E' $ | + id * id $ | E' → + T E' |
| + T E' $ | + id * id $ | 匹配 + |
| T E' $ | id * id $ | T → F T' |
| F T' E' $ | id * id $ | F → id |
| id T' E' $ | id * id $ | 匹配 id |
| T' E' $ | * id $ | T' → * F T' |
| ... | | |

### 舉例：BNF 表示法

Python 列表切片語法的 BNF：
```
<slice> ::= <expression> ":" <expression>
          | <expression> ":" <expression> ":" <expression>
          | <expression> ":" <expression> "[" <expression> "]"
          | <expression> ":" <expression> ":" <expression> "[" <expression> "]"
```

這展示了如何用 BNF 描述具體的語法規則。
