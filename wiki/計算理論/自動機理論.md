# 自動機理論：形式語言與計算模型

自動機理論研究抽象計算機器的數學模型，是形式語言、編譯器設計和計算複雜度的理論基礎。自動機從簡單的有限狀態機到複雜的圖靈機，構成了理解計算本質的層次結構。本頁介紹主要的自動機模型及其與正規語言、上下文無關語言的關係。

## 自動機與形式語言的關係

自動機理論的核心問題是：什麼樣的語言可以被機器識別？這個問題與「什麼可以被計算」密切相關。

## 有限狀態自動機

有限狀態自動機（Finite Automaton，FA）是最簡單的自動機模型，用於識別正則語言。它由有限數量的狀態和狀態之間的轉移組成。

### 確定性有限自動機 (DFA)

確定性有限自動機的特點是：每個狀態對每個輸入符號只有唯一確定的下一個狀態。

**形式定義**：DFA = (Q, Σ, δ, q₀, F)

- Q：有限狀態集合
- Σ：有限輸入字母表
- δ：轉移函數 Q × Σ → Q
- q₀：起始狀態
- F：接受狀態集合

**運作方式**：從起始狀態開始，讀取輸入字串的每個符號，根據轉移函數更新狀態。如果讀完所有輸入後處於接受狀態，則接受該字串。

**範例**：設計一個 DFA 識別包含子字串 "ab" 的所有字串。

狀態設計：
- q₀：起始狀態，還沒看到 "ab"
- q₁：剛讀到 'a'
- q₂：已讀到 "ab"（接受狀態）

轉移函數：
- δ(q₀, 'a') = q₁, δ(q₀, 'b') = q₀
- δ(q₁, 'a') = q₁, δ(q₁, 'b') = q₂
- δ(q₂, 'a') = q₂, δ(q₂, 'b') = q₂

接受狀態：F = {q₂}

### 非確定性有限自動機 (NFA)

非確定性有限自動機允許：一個狀態對同一個輸入有多個可能的轉移；空轉移（ε-轉移）不需要消耗輸入就能進行轉移。

**形式定義**：NFA = (Q, Σ, δ, q₀, F)

與 DFA 的主要區別是 δ : Q × Σ ∪ {ε} → P(Q)，返回一個狀態集合而非單一狀態。

**範例**：設計一個 NFA 識別以 "01" 結尾的二進位字串。

可以使用兩個並行的路徑：一個從頭開始直接檢查 "01"，另一個先處理前面的部分，最後汇合到檢查 "01" 的部分。

### DFA 與 NFA 的等價性

**定理**：對於任何 NFA，存在一個等價的 DFA 接受相同的語言。

**證明思路（子集構造法）**：
1. DFA 的每個狀態對應 NFA 狀態集合的一個子集
2. DFA 的起始狀態是 NFA 起始狀態的 ε-閉包
3. 對於狀態集合 S 和輸入 a，轉移到從 S 中任何狀態經過 a 可達的所有狀態的 ε-閉包
4. 如果狀態集合包含任何 NFA 接受狀態，則 DFA 的對應狀態為接受狀態

這個構造可能產生指數數量的狀態，但轉換總是可行的。

## 正則表達式

正則表達式是描述正則語言的另一種方式，與有限狀態機等價。

### 語法

正則表達式由以下元素組成：
- 基本符號：a, b, c...（表示字元）
- 空字串：ε
- 並聯：r | s（表示「或」）
- 串聯：rs（表示「連接」）
- 克萊尼星：r*（表示「零或多個」）

### 與有限狀態機的轉換

正則表達式可以轉換為 NFA（使用 Thompson 構造），NFA 可以轉換為 DFA（使用子集構造）。這個流程是編譯器詞法分析器的理論基礎。

### 範例

- "ab"：匹配字串 "ab"
- "a|b"：匹配 "a" 或 "b"
- "a*"：匹配零或多個 "a"
- "(a|b)*"：匹配任意數量的 a 和 b 的組合
- "a*b*"：匹配任意數量的 a 後跟任意數量的 b

## 下推自動機

下推自動機（Pushdown Automaton，PDA）比有限狀態機多了一個堆疊，這使其能夠識別上下文無關語言。

### 結構

```
┌─────────────────────────────────────────────────────────────┐
│                    下推自動機結構                           │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  ┌─────────┐      ┌─────────┐      ┌─────────────────────┐ │
│  │ 狀態控制 │─────▶│ 轉移函數 │─────▶│  無限堆疊           │ │
│  └─────────┘      └─────────┘      │ ┌───┬───┬───┐      │ │
│                                     │ │ Z │ X │ Y │ ... │ │ │
│                                     │ └───┴───┴───┴──────│ │
│                                     └─────────────────────┘ │
│                                                              │
│  能力：可以「記住」無限量的資訊（受限於堆疊結構）           │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 與上下文無關文法的對應

下推自動機與上下文無關文法是等價的：一個語言是上下文無關的，當且僅當有一個下推自動機能夠識別它。

這意味著，我們可以用文法來描述語言，用自動機來識別語言，兩種描述方式是等價的。

### 範例語言

下推自動機可以識別有限狀態機無法識別的語言，例如：
- {aⁿbⁿ | n ≥ 1}：n 個 a 後跟 n 個 b
- 匹配的括號：()()、(()())、(()())() 等

這些語言需要「計數」或「配對」的能力，有限狀態機做不到，但下推自動機的堆疊可以提供這種能力。

## 圖靈機

圖靈機是最強大的計算模型，可以識別所有可計算的語言。

### 結構

圖靈機由以下部分組成：
- 無限長度的磁帶（可以讀寫）
- 讀寫頭（在磁帶上移動）
- 有限狀態控制器
- 轉移函數

```

├─────────────────────────────────────────────────────────────┤
│                                                              │

│  │ 狀態寄存器 │───▶│ 轉移函數 │───▶│  無限磁帶              │ │
│  └─────────┘    └─────────┘    │ ┌───┬───┬───┬───┬───┐  │ │
│                                 │ │ □ │ a │ b │ c │ □ │  │ │
│                                 │ └───┴───┴───┴───┴───┘  │ │
│                                 │     ↑                  │ │
│                                 │   讀寫頭                │ │

│  能力：可以模擬任何其他計算模型                             │
```

## 自動機理論的應用

### 編譯器設計

編譯器的詞法分析階段使用有限狀態機識別 tokens，語法分析階段使用上下文無關文法描述語法，並使用下推自動機進行語法分析。

### 文本處理

正則表達式廣泛用於文本搜索、替換、驗證等任務，是許多程式語言和工具的標準功能。

### 協定驗證

網路協定的狀態可以用有限狀態機建模，用於驗證協定的正確性和發現潛在問題。

### 形式驗證

自動機理論是模型檢驗的基礎，用於驗證軟硬體系統的正確性。

## 複雜度與可計算性

自動機理論與計算複雜度理論密切相關：

- **可識別性**：有限狀態機識別正則語言，下推自動機識別上下文無關語言，圖靈機識別所有遞迴可枚舉語言
- **複雜度**：同一語言類別中的問題可能有不同的計算複雜度

## 舉例說明

### 舉例：正則表達式轉換為 NFA

正則表達式：a(b|c)*

Thompson 構造法：
1. a：建立 a 的 NFA
2. (b|c)*：建立 b|c 的 NFA，然後加 Kleene 星號

結果 NFA 結構：
```
q₀ --a--> q₁        q₂ --b--> q₃      q₄ --c--> q₅
              ↗↘           ↗↘              ↗↘
              ε   q₂       ε   q₃         ε   q₄
              ↙↖           ↙↖              ↙↖
              q₆ <--ε-- q₂₂              q₄₄
```

### 舉例：NFA 轉換為 DFA

NFA：
- 狀態：{0, 1, 2}
- 字母表：{a, b}
- 起始：{0}
- 接受：{2}
- 轉移：
  - 0 → {1, 2}（讀 a）
  - 1 → {2}（讀 b）

子集構造：
- S₀ = ε-閉包({0}) = {0}
- S₀ 讀 a → S₁ = ε-閉盖({1, 2}) = {1, 2}
- S₀ 讀 b → S₂ = ε-閉包({}) = {}
- S₁ 讀 a → S₃ = ε-閉包({}) = {}
- S₁ 讀 b → S₄ = ε-閉包({2}) = {2}

DFA 狀態圖：
```
{0} --a--> {1,2} --b--> {2}
   |                  |
   b                  a, b
   ↓                  ↓
  {}                  {}
```

### 舉例：下推自動機識別 aⁿbⁿ

語言：{aⁿbⁿ | n ≥ 1}

PDA 設計：
- 狀態：q₀, q₁, q₂
- 堆疊符號：Z（底部標記）, A
- 起始狀態：q₀
- 接受狀態：q₂

轉移：
1. q₀, a, Z → q₀, AZ（讀 a 入堆疊）
2. q₀, a, A → q₀, AA（讀 a 入堆疊）
3. q₀, b, A → q₁, ε（讀 b 彈出 A）
4. q₁, b, A → q₁, ε（繼續彈出）
5. q₁, ε, Z → q₂, Z（堆疊空，接受）

執行示例（輸入 "aabb"）：
- 讀 a a：堆疊變成 A A
- 讀 b：彈出 A
- 讀 b：彈出 A，堆疊剩 Z
- 進入接受狀態 q₂

### 舉例：自動機理論的實際應用

正則表達式匹配 URL：

模式：https?://[a-zA-Z0-9.-]+(/[a-zA-Z0-9./_-]*)

實際應用：
- 詞法分析器將正則表達式轉為 NFA
- NFA 轉為 DFA（確定化）
- DFA 最小化
- 產生高效的正則表達式引擎

這就是 grep、sed 等工具的原理。
