行程與執行緒

並發執行的基本單位

行程（Process）

行程是正在執行的程式實例，是作業系統資源分配的基本單位。每個行程擁有獨立的位址空間、程式計數器、堆疊和系統資源。

行程狀態

  • 新建（New）：行程正在被建立
  • 就緒（Ready）：等待 CPU 排程
  • 執行（Running）：正在 CPU 上執行
  • 等待（Waiting）：等待 I/O 或事件
  • 終止（Terminated）：執行完畢

執行緒（Thread）

執行緒是行程內部的輕量級執行單位，是 CPU 排程的基本單位。同一行程內的多個執行緒共享位址空間和資源，但各自擁有獨立的堆疊和暫存器狀態。

執行緒的優勢

  • 共用記憶體：直接共享資料，無需 IPC
  • 快速切換：開銷遠小於行程切換
  • 高效率：建立和終止成本低

行程間通訊（IPC）

  • 管道（Pipe）：單向資料流
  • 訊息佇列（Message Queue）：非同步傳遞
  • 共享記憶體（Shared Memory）：高效共享
  • 信號（Signal）：非同步事件通知
  • Socket：網路通訊
