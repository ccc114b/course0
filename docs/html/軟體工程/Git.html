Git

分散式版本控制系統

核心概念

  • Repository（倉庫）：存放專案和所有歷史的地方，分為本地和遠端倉庫
  • Commit（提交）：對專案某一時刻的「快照」，每個 commit 有唯一的 SHA-1 雜湊值
  • Branch（分支）：獨立開發的程式碼線，預設分支通常是 main 或 master
  • Merge（合併）：將一個分支的變更合併到另一個分支
  • HEAD：指向當前 checkout 的 commit

Branch 與 Merge

Git 的分支是輕量級指標，指向特定 commit。常見操作：

  • git branch feature-x — 創建分支
  • git checkout feature-x — 切換分支
  • git merge feature-x — 合併分支到當前分支
  • git rebase main — 將當前分支基底移到 main 的最新 commit

協作工作流程

  • Fork & Pull Request：開源協作模式，fork → 修改 → PR → 審查 → 合併
  • Git Flow：適合釋出版本較固定的專案，包含 develop、release、hotfix 分支
  • GitHub Flow：任何修改都從 main 開分支，PR 審查後合併回 main

常用指令

# 配置 git config --global user.name "Your Name" git config --global user.email "email@example.com" # 基本操作 git init git add . git commit -m "message" git push origin main git pull origin main # 查看 git status git log --oneline --graph git diff
