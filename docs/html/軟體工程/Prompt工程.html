Prompt 工程

設計和最佳化輸入提示詞，提升 LLM 輸出品質

提示詞基本結構

  • System：角色定義、能力邊界
  • Context：背景資訊、上下文
  • Examples：範例（Few-shot）
  • Task：具體任務
  • Format：輸出格式

核心技巧

Zero-shot Prompting

不提供任何範例，直接要求模型執行任務。適用於簡單任務。

Few-shot Prompting

提供 1-多個範例，讓模型學習模式。範例越接近目標任務效果越好。

Chain-of-Thought（CoT）

引導模型逐步推理，適合需要多步驟思考的複雜任務，如數學問題、邏輯推理。

進階技術

  • Role-playing：賦予 LLM 特定角色（專家、教師等）
  • Template Patterns：使用模板結構化提示詞
  • Context Engineering：精心組織上下文資訊
  • Harness Engineering：控制 LLM 的執行環境和工具使用

Prompt 範例

# System 你是一個專業的 Python 程式設計師， 擅長寫高品質、可維護的程式碼。 # Task 請寫一個函式，使用二分搜尋法在有序陣列中尋找目標值。 # Format - 回傳目標值的索引，找不到則回傳 -1 - 包含型別註記 - 包含時間複雜度註解
