軟體工程

軟體開發流程與最佳實踐

開發流程

從需求分析到軟體交付，涵蓋傳統瀑布模型與現代敏捷開發方法論。

  • 軟體生命週期 — 需求分析、設計、實作、測試、部署、維護
  • 需求工程 — 用戶故事、功能與非功能需求、需求驗證
  • 敏捷開發 — Scrum、看板、極限程式設計（XP）
  • 版本控制 — 集中式 vs 分散式、版本控制策略
  • Git — 核心概念、分支策略、協作工作流程

品質保證

透過測試、程式碼品質管控與技術債管理來確保軟體品質。

  • 單元測試 — 測試金字塔、TDD、測試框架
  • 程式碼品質 — 可讀性、可維護性、重構、技術債
  • 設計模式 — 創建型、結構型、行為型模式

架構設計

高層次系統設計，從單體到微服務的架構演進。

  • 軟體架構 — 單體、分層、微服務、事件驅動架構
  • 微服務架構 — 服務拆分、Saga、API 閘道
  • 領域驅動設計 — 實體、值對象、聚合、限界上下文

DevOps 與工具

將開發與維運整合，實現持續整合、持續部署與自動化。

  • DevOps — 文化、自動化、監控、持續改進
  • CI/CD — 持續整合、持續部署、Pipeline 設計
  • Docker — 容器化、Image、Container 管理
  • Prompt 工程 — LLM 提示詞設計、Few-shot、CoT
