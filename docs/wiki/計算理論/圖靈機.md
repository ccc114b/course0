# 圖靈機 (Turing Machine)

**標籤**: #圖靈機 #Turing #計算理論 #停機問題 #可計算性
**日期**: 2024-01-15

圖靈機是艾倫·圖靈（Alan Turing）在 1936 年提出的抽象計算模型，被認為是電腦科學的理論基石。圖靈機不僅解決了判定問題，也定義了「可計算性」的數學意義，是所有現代電腦的理論原型。

## 歷史背景

1936 年，數學家大衛·希爾伯特（David Hilbert）提出了著名的「判定問題」（Entscheidungsproblem）：是否存在一個演算法可以判定任意數學命題的真假？

圖靈在論文《論可計算數及其在判定問題中的應用》中提出了圖靈機模型，並證明了不存在這樣的演算法——這就是著名的停機問題。這一工作與阿隆佐·丘奇的 λ 演算並行發展，兩者被證明是等價的，共同構成了 Church-Turing 論點的基礎。

## 形式定義

圖靈機是一個七元組 M = (Q, Σ, Γ, δ, q₀, q_accept, q_reject)，各元件的意義如下：

| 符號 | 意義 |
|------|------|
| Q | 有限的非空狀態集合 |
| Σ | 輸入字母表（不含空白符號） |
| Γ | 磁帶字母表（包含 Σ 和空白符號 □） |
| δ | 轉移函數 Q × Γ → Q × Γ × {L, R} |
| q₀ | 起始狀態，q₀ ∈ Q |
| q_accept | 接受狀態，q_accept ∈ Q |
| q_reject | 拒絕狀態，q_reject ∈ Q |

## 轉移函數

轉移函數 δ 定義了圖靈機的核心行為：

```
δ(q, a) = (p, b, L/R)

含義：當處於狀態 q、讀到符號 a 時，
      轉移到狀態 p，寫入符號 b，向左(L)或向右(R)移動
```

例如，轉移 ('q0', '0') = ('q1', 'X', 'R') 表示：當處於 q0 狀態、讀到 '0' 時，轉移到 q1 狀態，將磁帶上的 '0' 改為 'X'，然後向右移動一個位置。

## 圖靈機 vs 有限狀態機

| 特性 | 有限狀態機 (FSM) | 圖靈機 |
|------|------------------|--------|
| 磁帶 | 無 | 無限 |
| 寫入能力 | 否 | 是 |
| 移動方向 | 只能前進 | 左/右/靜止 |
| 記憶能力 | 有限狀態 | 無限（磁帶） |
| 計算能力 | 正規語言 | 任何可計算問題 |
| 可識別語言 | 正則語言 | 遞迴可枚舉語言 |

圖靈機比有限狀態機更強大之處在於：它可以寫入和修改磁帶，這使其具有了真正的「記憶能力」，能夠解決有限狀態機無法解決的問題，如識別語言 {aⁿbⁿ | n ≥ 1}。

## 範例：識別 aⁿbⁿ 語言

讓我們考慮如何設計一個圖靈機來識別語言 {aⁿbⁿ | n ≥ 1}，即任意數量的 a 後跟相同數量的 b。

狀態設計：
- q0：起始狀態，找尋 a
- q1：找到 a，標記並向右尋找 b
- q2：找到對應的 b，標記並向左尋找下一個 a
- q3：成功完成，準備接受
- q4：失敗狀態

轉移邏輯：
1. 從 q0 出發，將第一個 'a' 改為 'X'，進入 q1
2. 在 q1 狀態下向右移動，跳過 'a' 直到找到 'b'
3. 將找到的 'b' 改為 'Y'，進入 q2
4. 在 q2 狀態下向左移動，跳過 'a' 和 'Y' 直到找到 'X'
5. 回到 'X' 旁邊的 'a'，重複步驟 1-4
6. 當所有 'a' 都處理完畢且沒有 'b' 時，進入接受狀態 q3

這個範例展示了圖靈機如何通過在磁帶上標記和回溯來識別需要「配對」的語言。

## 停機問題

停機問題是圖靈機理論中最著名的結果，也是計算機科學中最重要的不可判定問題。

### 問題陳述

停機問題詢問：是否存在一個圖靈機 H，能夠判斷任意圖靈機 M 在任意輸入 w 上是否會停止？

形式化地說：是否存在一個圖靈機 H，接受輸入 (M, w)，當 M 在 w 上停機時接受，當 M 在 w 上不停機時拒絕？

### 證明（對角線論證）

證明使用反證法。假設存在這樣的圖靈機 H，我們可以構造一個悖論機器 P：

```
構造 P 如下：
當輸入是 <P> 時：
  如果 H(<P>, <P>) 說「會停機」
    则进入无限循环
  否則
    停止
```

現在問：P(P) 會停機嗎？

- 如果 H 說 P(P) 會停機，則 P 會進入無限迴圈，不会停机
- 如果 H 說 P(P) 不會停機，則 P 會停止

無論哪種情況都產生矛盾。因此，這樣的 H 不可能存在。

### 意義

停機問題的意義深遠：
- 存在無法用演算法解決的問題
- 電腦有本質上的限制
- 數學真理不等於可計算真理
- 我們不能寫出一個完美的程式來分析所有程式

## Church-Turing 論點

Church-Turing 論點是計算理論中最基本的假設：

> 任何可有效計算的函數都可以由圖靈機計算。

這不是一個定理，而是一個「論點」（thesis），因為「可有效計算」這個概念本身並非數學定義。論點得到以下支持：

1. 圖靈機可以模擬任何已知的計算模型
2. 所有提出用於描述「可計算性」的數學模型都被證明與圖靈機等價
3. 沒有任何已知的計算問題是圖靈機無法解決但其他模型可以解決的

Church-Turing 論點的意義在於：它定義了「可計算」的數學意義，從而使我們能夠討論問題的「可解決性」。

## 通用圖靈機

圖靈機的另一個重要概念是「通用圖靈機」——一個可以模擬任何其他圖靈機的圖靈機。

通用圖靈機接受兩個輸入：
1. 任意圖靈機 M 的描述（編碼）
2. 任意輸入 w

它模擬 M 在 w 上的行為，相當於現代電腦中「軟體」的概念。這預示了存儲程式電腦的設計原理。

## 複雜度理論

圖靈機不僅用於定義可計算性，也用於定義計算複雜度：

| 複雜度類別 | 定義 |
|-----------|------|
| P | 可在多項式時間內由確定性圖靈機解決 |
| NP | 可在多項式時間內由非確定性圖靈機解決 |
| PSPACE | 可在多項式空間內由確定性圖靈機解決 |
| EXPTIME | 可在指數時間內由確定性圖靈機解決 |

## 舉例說明

### 舉例：圖靈機識別 aⁿbⁿ 語言

語言 L = {aⁿbⁿ | n ≥ 1}，例如 "ab", "aabb", "aaabbb"。

設計思路：
1. 將 a 逐一改為 X，向右尋找對應的 b
2. 將找到的 b 改為 Y，向左回溯
3. 重複直到所有 a 都處理完
4. 檢查所有 b 都已變為 Y

狀態轉移示例：
- 狀態 q0：讀到 a，變為 X，右移
- 狀態 q1：讀到 a 繼續右移，讀到 b 變為 Y，左移
- 狀態 q2：讀到 X 右移，讀到 Y 左移，回到開頭

輸入 "aabb" 的執行過程：
```
_ a a b b _
  ↑
q0 讀 a，變 X，右移

_ X a b b _
    ↑
q1 讀 a，右移

_ X a b b _
      ↑
q1 讀 b，變 Y，左移

_ X a Y b _
    ↑
q2 讀 a，右移

_ X a Y b _
  ↑
q2 讀 X，右移

_ X a Y b _
↑
q0 讀 X，右移找下一個 a
...（重複直到完成）
```

### 舉例：通用圖靈機的概念

通用圖靈機 U 接受輸入 (M, w)：
- M：任意圖靈機的描述（編碼）
- w：任意輸入字串

U 的行為：模擬 M 在 w 上的執行。

這相當於現代電腦的「軟體」概念——同一台硬體（通用圖靈機）可以透過載入不同的程式（M）來執行不同的任務。

### 舉例：停機問題的實際意義

為什麼停機問題重要？

假設有一個程式分析工具 Halt(P) 可以判斷 P 是否會停機。

這意味著我們可以：
- 檢測無窮迴圈
- 驗證程式是否會崩潰
- 證明數學猜想

但實際上：
```
病毒程式 V(V):
    if Halt(V) then
        無限複製自己
    else
        停止
```

如果 V 會停機，Halt 會預測「會停機」，但 V 會進入無窮迴圈。
如果 V 不會停機，Halt 會預測「不會停機」，但 V 會停止。

無論如何都會出錯。因此，這樣的工具不可能存在。
