# 函數式編程 (Functional Programming)

**標籤**: #函數式編程 #FunctionalProgramming #Lambda演算 #高階函數 #不可變性 #純函數
**日期**: 2024-01-15
**更新**: 2026-04-27

函數式編程（Functional Programming，FP）是一種編程範式，其核心思想是將計算視為函數的求值過程，並避免改變共享狀態和可變資料。函數式編程強調不可變性、第一類函數、聲明式編程和高階函數。這種編程範式在近年來隨著多核處理器和分散式系統的普及而變得越來越重要。

## 歷史背景

### 理論基礎

函數式編程的理論基礎可以追溯到 1936 年，當時阿隆佐·丘奇（Alonzo Church）發明了 Lambda 演算，這是函數式編程的數學基礎。Lambda 演算是一種形式系統，用於描述函數和函數應用，它與圖靈機被證明是計算等價的——這就是著名的 Church-Turing 論點。

1920 年代，Moses Schönfinkel 和 Haskell Curry 發展了組合子邏輯，這是另一種影響函數式編程的數學邏輯系統。Haskell 語言的名稱就是為了紀念 Haskell Curry。

### 語言發展

1958 年，約翰·麥卡錫（John McCarthy）創建了 Lisp 語言，這是第一個廣泛使用的函數式語言。Lisp 引入了很多至今仍在使用的函數式編程概念，包括條件表達式、遞迴和高階函數。

1978 年，羅伯特·米爾納（Robin Milner）創建了 ML 語言，這是第一個帶有靜態類型系統的函數式語言，後來催生了 OCaml 和 F# 等語言。

1990 年，Haskell 語言誕生，它採用惰性求值和純函數式設計，對函數式編程的發展產生了深遠影響。

2000 年代以來，主流語言開始引入函數式特性，包括 Scala、Clojure、F# 等。2010 年代，函數式編程經歷了復興，React、Redux、RxJS 等庫在 Web 開發中广泛使用函數式概念。

## 核心概念

### 一等公民（First-Class Functions）

在函數式語言中，函數是「一等公民」，這意味著函數可以作為參數傳遞、作為返回值、賦值給變數，以及存儲在資料結構中。這種特性使得高階函數成為可能，允許我們編寫更具抽象性的代碼。

函數作為一等公民的能力是函數式編程的基礎，它使我們能夠創建通用的函數組合工具、映射和過濾操作，以及各種函數式設計模式。

### 不可變性（Immutability）

不可變性是函數式編程的核心原則之一。在函數式編程中，資料一旦創建就不能被修改。與其在原資料上進行修改，我們創建新的資料結構來容納結果。

不可變性帶來了多重好處：它消除了共享狀態引起的 bug；它使並行編程變得更加安全，因為不需要擔心資料競爭；它也使得程式行為更容易理解和推斷，因為函數不會產生意外的副作用。

### 純函數（Pure Functions）

純函數的輸出只依賴於其輸入參數，不會產生副作用。純函數不會修改全域狀態、不會進行 IO 操作、也不會拋出異常。相同的輸入總是產生相同的輸出，這使得純函數易於測試、調試和並行化。

與純函數相對的是非純函數，它們可能修改全域狀態依賴於外部環境，或產生不可預測的行為。函數式編程傾向於將副作用隔離到程式碼的特定部分，以便更好地控制和理解程式行為。

## 高階函數

高階函數是接受函數為參數或返回函數的函數。這些函數是函數式編程的基本構建模組。

### Map（映射）

Map 操作對集合中的每個元素應用一個函數，產生一個新集合。這是函數式編程中最常見的操作之一，允許我們描述「對每個元素做某件事」而不是編寫顯式的循環。

例如，對一個數字列表中的每個元素進行平方運算，使用 Map 操作可以優雅地表达为：將平方函數應用於列表中的每個元素。

### Filter（過濾）

Filter 操作根據謂詞函數的結果篩選集合中的元素，保留滿足條件的元素，移除不滿足條件的元素。這使得我們可以用聲明式的方式表达「選擇符合條件的元素」。

### Reduce（歸約）

Reduce 操作（也稱為 fold）將集合中的所有元素歸約為單一值。這個過程通過反覆應用一個二元運算符來實現，從初始值開始，逐步組合集合中的元素。

Reduce 是實現各種集合操作的強大工具，包括求和、求積、連接字串、尋找最大值等。

### Compose（組合）

函數組合是將多個函數組合成一個新函數的過程。如果我們有函數 f 和 g，f ∘ g 表示「先應用 g，然後應用 f」。函數組合允許我們從簡單的函數構建複雜的運算，同時保持代碼的簡潔和可讀性。

### Curry 化（Currying）

Curry 化是將多參數函數轉換為單參數函數鏈的過程。例如，一個接受兩個參數的函數 f(a, b) 可以轉換為接受第一個參數並返回一個接受第二個參數的函數 g(a)(b)。這種轉換使函數更加靈活，可以部分應用，創建新的函數。

## 常見模式

### 函數式資料結構

函數式語言通常使用不可變資料結構。這些資料結構在被「修改」時，其實是創建了一個新版本，而原始版本保持不變。持續資料結構（Persistent Data Structures）是一種特殊的資料結構，它們的所有版本都可以被高效地共享。

函數式列表是最簡單的不可變資料結構之一，通常由「頭」和「尾」組成。頭是列表的第一個元素，尾是剩餘元素的列表。追加元素不是修改原列表，而是創建一個新列表，其頭是新元素，尾是原列表。

### Option/Maybe 型態

Option 型態（又稱 Maybe）用於表示可能存在也可能不存在的值。它有兩個變體：Some（或 Just）表示值存在，None（或 Nothing）表示值不存在。這種設計強制程式處理可能缺失的情況，避免空指標異常。

Option 型態配合 map、flatMap 等函數使用，可以優雅地處理連續的可選值操作，而無需使用空值檢查。

### Either 型態

Either 型態通常用於表示兩種可能的結果：一種表示成功（通常稱為 Right），另一種表示失敗（通常稱為 Left）。這種設計在錯誤處理中非常有用，允許函數返回有意義的錯誤訊息而不是拋出異常。

Either 型態支持與 Option 類似的操作，包括 map 和 flatMap，但區分成功和失敗的情況。這使得錯誤傳播變得更加明確和可控。

## 著名函數式語言

### Haskell

Haskell 是最純粹的函數式語言之一，採用惰性求值和純函數式設計。Haskell 沒有任何命令式特性，所有函數都是純函數。惰性求值意味著表達式只有在需要時才會被求值，這使得 Haskell 能夠處理無窮資料結構和優雅地實現各種優化。

Haskell 的類型系統也非常強大，包括代數資料類型、類別（typeclass）和模態類型。它也是單子（Monad）概念的流行語言，單子提供了一種結構化處理副作用的方式。

### Lisp 方言

Lisp 是歷史最悠久的函數式語言之一，其簡單而强大的 S-表達式語法對後來的語言產生了深遠影響。Common Lisp 是 Lisp 的標準化版本，結合了函數式和命令式特性。Clojure 是 Lisp 在 JVM 上的現代實現，強調不可變性和函數式編程。

### Scala

Scala 是一種結合函數式和物件導向的語言，運行在 JVM 上。它提供了完整的函數式特性，包括高階函數、不可變資料結構、模式匹配和 For comprehension。Scala 的設計目標是既要保持與 Java 的兼容性，又要提供更強大的抽象能力。

### F#

F# 是 .NET 平台上的函數式語言，與 C# 和 VB.NET 共用運行時。F# 結合了函數式和命令式特性，其類型系統支持推理和模式匹配。管道運算子是 F# 的特色功能，允許以流式方式組合函數。

## 主流語言的函數式特性

現代主流語言越來越多地引入函數式特性。JavaScript 支持箭頭函數、Map/Filter/Reduce、Promise 和 async/await 等函數式概念。Python 支援 Lambda 表達式、內建高階函數、列表推導式和裝飾器。Rust 作為系統程式語言，也提供了迭代器適配器、閉包和 Result/Option 類型等函數式特性。

## 函數式與命令式比較

函數式編程與命令式編程在多個方面存在差異。在狀態管理方面，函數式傾向於不可變資料，而命令式使用可變狀態。在控制流方面，函數式依賴函數調用，而命令式使用語句和循環。在執行順序方面，函數式代碼可以具有非確定性，而命令式代碼通常是確定性的。

函數式編程的優勢包括：更高的可讀性和可維護性，因為代碼更具聲明性；更容易並行化，因為不可變資料不存在資料競爭；更強的組合性，高階函數便於重用；以及更容易靜態分析和優化。

劣勢包括：學習曲線較陡，抽象概念較難掌握；有時效能較差，惰性求值和不可變資料有額外開銷；調試時棧追蹤可能更複雜；以及團隊可能不熟悉函數式模式。

## 舉例說明

### 舉例：Map、Filter、Reduce 的使用

假設有列表：[1, 2, 3, 4, 5]

Map（每個元素平方）：
[1, 4, 9, 16, 25]

Filter（只保留偶數）：
[2, 4]

Reduce（求和）：
15

組合使用（求平方和）：
[1, 4, 9, 16, 25] 的總和 = 55

### 舉例：Curry 化

未 Curry 化：
add(a, b, c) = a + b + C

Curry 化後：
add = curry(add)
add(1)(2)(3) = 6
add(1, 2)(3) = 6
add(1)(2, 3) = 6

好處：可以「固定」部分參數
add10 = add(10)
add10(5) = 15

### 舉例：不可變資料結構的優勢

命令式（可變）：
list = [1, 2, 3]
list.append(4)
# list 現在是 [1, 2, 3, 4]

函數式（不可變）：
list = [1, 2, 3]
new_list = list + [4]
# list 仍是 [1, 2, 3]
# new_list 是 [1, 2, 3, 4]

並行優勢：在多執行緒環境下，不可變資料不需要同步機制，因為沒有人能修改它。

### 舉例：Option 型態處理空值

假設要從資料庫取得用戶，若用戶不存在則返回預設值。

傳統方式：
```
user = getUser(id)
if user != null:
    return user.name
else:
    return "匿名"
```

函數式方式（使用 Option）：
```
getUser(id)
    .map(lambda u: u.name)
    .getOrElse("匿名")
```

好處：不用檢查 null，程式碼更簡潔，且在編譯時就能發現可能的空值問題。

### 舉例：純函數的測試優勢

純函數測試簡單：
```
assert add(2, 3) == 5
assert add(0, 5) == 5
assert add(-1, 1) == 0
```

每次呼叫都會返回相同結果，不需要 Mock 物件或設置測試環境。

非純函數測試困難：
```
counter = 0
def increment():
    global counter
    counter += 1
    return counter
```

測試結果依賴呼叫次數，需要重置狀態。
